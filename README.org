#+title: Learning Org Repository

这个仓库是你学习资料的 =org 索引层= ，不是原始资料存储层。

* 设计原则
- 只保存 =org= 笔记、索引、总结、问题清单和学习计划。
- 大体积资料继续留在原目录，例如课件、PDF、zip、实验代码、嵌套 Git 仓库。
- 每个正在学习的主题只保留一个 =index.org= 作为入口页。
- 入口页负责三件事：说明学习目标、链接原始目录、沉淀真正有价值的总结。

* 目录约定
- =00-index/= 全局入口、活动主题、临时收集箱。
- =10-courses/= 学校课程或明显带有课程节奏的内容。
- =20-topics/= 长期技能主题，例如语言、协议、框架、工具。
- =30-projects/= 和学习主题绑定的练习项目。
- =40-resources/= 粗粒度资源索引，只记录“资源在哪里”，不再细拆。
- =90-archive/= 已暂停或阶段性完成的内容。

* 建议工作流
1. 临时想法、问题、摘录先记到 [[file:00-index/inbox.org][00-index/inbox.org]]。
2. 每周把 inbox 里真正有价值的内容整理到对应主题的 =index.org=。
3. 主题页里只保留你以后还会回看的结构化内容，不重复抄课件。
4. 一个主题如果开始有稳定产出，再考虑拆出 =concepts.org= 或 =practice.org=。
