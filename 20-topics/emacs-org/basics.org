#+title: Emacs org Basics

* Org Mode 基础: 结构、设计原理

** 本节定位
Org不是一个排版语言，而是一种在Emacs里操作树状文本的方式。

官方文档在开头就强调，Org是建立在 Outline Mode 之上的；它是用纯文本保存内容，用折叠与结构编辑来管理大型文件。也就是说，Org 模式的核心是“可组织、可折叠、可继续加工”。

-----

** 为什么会有 Org: 它解决什么问题

Org 关心的是 “怎么长期维护笔记”。

当一个文件越来越长时，最麻烦的是：
- 找不到结构
- 无法只看局部
- 很难移动大块内容
- 任务、笔记、链接、代码彼此割裂


Org 对此的解决方法是：把文档看成层级结构，也就是 outline。官方文档将 Org 描述为 outliner；通过 folding(折叠)，可以只看当前分支，或看整棵子树。TAB对应的 ~org-cycle~ 就是这种设计的核心接口。

所以，Org 最开始就是一种“组织内容”的方法，之后才产生了 ~TODO~ 、 ~Agenda~ 、 表格、代码块这些能力。

-----

** Org 的设计原理: 纯文本 + 层次结构 + 结构操作

*** 纯文本

Org 文件本质上是普通文本文件，用任何编辑器都能打开，不依赖私有数据库，也不依赖专门的二进制格式。官方手册把这一点放得很前面：它是 plain-text system。

这意味着几个很重好的结果：
- 文件可长期保存，不容易被工具绑定
- 适合 Git 管理
- 可以和 shell、grep、脚本、版本控制一起工作
- 即便离开 Emacs，内容也依然存在


但“纯文本”不等于“没有结构”，Org 的关键是：它用一套非常轻量的标记，把纯文本组织成层级化对象。
