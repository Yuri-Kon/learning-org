#+title: Org 进阶基础
* Org 进阶基础：Sparse Trees, Lists, Drawers 与 Blocks
** 本节定位

这一节仍然属于 Org 的基础层，不进入任务系统，也不进入 Agenda。

它解决四个问题：

- 当文件很长时，我如何只看我现在想看的内容
- 在一个 headline 内，如何组织局部内容
- 如何保存不想总是看见的附加信息
- 如何在 Org 中容纳“特殊结构文本”


学完这一节后，对 Org 的理解会从 “会折叠标题” 进入到 “会组织节点内部内容”

-----
** Sparse Trees: 只显示要找的部分
*** 它是什么

官方把 sparse tree 视为 Org 的一个很重要的特性：它会尽可能把整个文档折叠起来，只显示被选中信息，以及这些信息上方必要的标题层级。

这和普通搜索不一样。普通搜索只找到“匹配文本”；而 Sparse Tree 是“以树的结构方式显示匹配结果”。

所以它非常适合长文档。不是在一坨搜索结果里来回跳转，而是在原文档结构里只看有用节点。

*** 为什么它重要
这其实体现了 Org 的核心设计：

你不是单纯操作一份线性文档，而是在一棵树上做筛选。

Sparse tree 的价值在于：

- 保留上下文层级
- 隐藏无关部分
- 让搜索结果仍然处于原来的结构位置上


这就是 Org 的“搜索也按照结构来”思路。

*** 现在最该会的命令

- ~C-c /~: 进入 sparse tree 调度入口
- ~C-c / r~: 按正则搜索并生成 sparse tree
- ~M-g n~: 跳转到下一个匹配
- ~M-g p~: 跳转到上一个匹配

~org-occur~ 会显示匹配项；如果匹配在标题中，就显示标题；如果匹配在正文中，就先死后标题和正文，并且保留上方的层级结构

*** 应当建立的理解

Sparse Tree 不是“另一个视图文件”，而是当前buffer的另一种结构化筛选。

它和之前学习的 folding 是一脉相承的，只不过这次不是手动按层折叠，而是按条件自动暴露相关节点。

** Plain Lists: headline 之下的局部结构

*** 它是什么

官方手册里说，列表是 outline tree 里某个 entry 里面提供附加结构的方式，并且也为 checkbox 提供基础。Org 支持 unorderer list, orderer list 和 description list。

这句话说明了：

- list 不和 headline 竞争
- list 是headline 内部的局部结构
