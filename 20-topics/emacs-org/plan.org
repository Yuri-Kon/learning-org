#+TITLE: Org Mode 系统学习计划(Learing Plan)

* 学习计划

** 计划说明

这份计划的目标是逐步建立以下三种能力：

- 能把 Org 当作结构化写作工具使用
- 能把 Org 当作任务与知识管理工具使用
- 能理解 Org 的核心设计逻辑，并知道各模块之间如何相互连接


学习顺序遵循一个基本原则：先学习结构，再学习任务，再学习组织，再学习扩展。

-----

** 总体学习目标

*** 目标1：理解 Org 的核心模型

- [X] 理解 Org 建立在 Outline 结构之上
- [X] 理解 headline, subtree, plain list, drawer 等对象
- [X] 理解 “操作结构” 而非 “操作文本” 的思维方式


*** 目标2：掌握日常使用所需的基础能力

- [X] 能独立编写结构清晰的 Org 文件
- [X] 熟练使用折叠、导航、结构编辑
- [X] 能正确区分标题、列表、正文、属性、代码块、链接


*** 目标3：任务管理能力

- [ ] 掌握 TODO 状态、时间戳、调度、截止日期
- [ ] 掌握 Agenda的基本工作方式
- [ ] 能用 Org 管理学习与项目任务


*** 目标4：建立知识组织能力

- [ ] 能设计自己的 Org 文件结构
- [ ] 能使用链接、ID、引用组织知识
- [ ] 能在一个长期仓库中维护index、topic、project能文件


*** 目标5：进入 Org 的进阶使用阶段

- [ ] 掌握代码块、表格、导出
- [ ] 理解 Babel 的基本用途
- [ ] 能根据需要扩展到 org-capture, org-reflie, org-roam 等

*** Phase 1: 基础结构(Basics)

目标：
- [X] 理解 Org 的本体是树结构

- Document Structure
- Headline
- Visibility Cycling
- Motion
- Structure Editing
- Plain Lists
- Drawers


应掌握的能力：

- [X] 能手写一个三层结构文件
- [X] 能用 TAB / S-TAB 在不同可见性状态之间切换
- [X] 能用结构编辑命令整体移动一个子树
- [X] 能区分标题与列表的作用
- [ ] 能知道属性抽屉是什么，放在哪里


实践内容：

- [X] 编写 basics.org
- [X] 制作一个 Emacs 学习树

*** Phase 2: 标记与文本对象

- 熟悉正文内部的常见对象

- 强调、等宽文本、引用等基础标记
- 段落与换行
- 链接
- 脚注
- 块（example / quote / src 等）的基本形态

应掌握的能力：

- [ ] 会写内部链接、文件链接、外部链接
- [ ] 会区分普通正文与 block
- [ ] 会使用 example block 保留原样文本
- [ ] 会使用 src block 表示代码内容


实践任务：

- [ ] 在 basics.org 中补充链接与文本对象示例
- [ ] 编写一个 links-and-blocks.org 练习文件
- [ ] 为学习索引加入文件间跳转链接



通过标准：
- 能在一个 Org 文件中组织“正文 + 引用 + 示例 + 链接”而不混乱

- 状态切换

应掌握的能力：

- [ ] 会创建 TODO / DONE 项目
- [ ] 会切换状态
- [ ] 知道 headline TODO 和 checkbox 的区别
- [ ] 会给任务打 tag 和 priority


实践任务：

- [ ] 把学习计划中的阶段条目改造为 TODO 项目
- [ ] 给学习任务设定状态

通过标准：

- 能用 Org 管理一周内的学习事务

-----

** 推荐学习方式

建议按照以下顺序推进：

1. basics
2. links-and-blocks
3. todo
4. dates-and-times
5. agenda
6. file-organization
7. tables-and-export
8. babel
9. capture / refile / archive
10. 个性化工作流


原因：

- basics 决定是否能真正理解 Org
- links / blocks 能正常写文档
- TODO + 时间 + Agenda 才能组成任务系统
- 文件组织决定能否长期维护
- 导出和Babel属于变强


-----

** 每一节的固定学习模板

*** 1. 本节定位
这节属于整个 Org 体系中的哪一部分

*** 2. 官方文档对应章节
记录本节对应官方文档位置

*** 3. 核心概念
解释这一节在 Org 中真正解决什么问题

*** 4. 设计原理
说明它为什么这样设计，和前后模块有什么关系

*** 5. 基础语法或核心命令
列出最必要的语法和操作

*** 6. 底层原理
说明它在 Org 的结构中属于什么对象，与 headline, subtree, drawer, properties, buffer 的关系是什么

*** 7. 最小掌握集
写清楚“学完这一节必须会什么”

*** 8. 练习
要求自己手写示例，进行实际操作

*** 9. 常见混淆点
记录容易混淆的概念

*** 10. 本节总结
用自己的话重写一遍核心认识
