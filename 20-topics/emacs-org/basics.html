1. Org Mode 基础: 结构、设计原理

1.1. 本节定位

Org不是一个排版语言，而是一种在Emacs里操作树状文本的方式。

官方文档在开头就强调，Org是建立在 Outline Mode 之上的；它是用纯文本保存内容，用折叠与结构编辑来管理大型文件。也就是说，Org 模式的核心是“可组织、可折叠、可继续加工”。

1.2. 为什么会有 Org: 它解决什么问题

Org 关心的是 “怎么长期维护笔记”。

当一个文件越来越长时，最麻烦的是：

  • 找不到结构
  • 无法只看局部
  • 很难移动大块内容
  • 任务、笔记、链接、代码彼此割裂

Org 对此的解决方法是：把文档看成层级结构，也就是 outline。官方文档将 Org 描述为 outliner；通过 folding(折叠)，可以只看当前分支，或看整棵子树。TAB对应的 org-cycle 就是这种设计的核心接口。

所以，Org 最开始就是一种“组织内容”的方法，之后才产生了 TODO
