-----
* 为什么黑盒测试不能穷举
黑盒测试把系统当作黑盒，只看功能实现，不看内部逻辑；但如果直接穷举输入，工作量会大到不可能完成。

黑盒测试设计方法的本质：

*从大量可能输入中，挑选出“最具代表性”的少量测试数据*。

- 黑盒测试关注的是 *输入-输出* 关系
- 不是要测完所有数据，而是用尽可能少的数据覆盖尽可能多的情况


-----
* 等价类划分法
** 等价类是什么
等价类是某个输入域的子集，在这个子集中每个输入数据的作用是等效的。设计要同时考虑有效等价类和无效等价类。

比如 “成绩0-100”这个输入条件，表面上有101个数据，但是从等价类角度可以先分为三类：

- 合法：0~100
- 非法：小于0
- 非法：大于100


-----
** 为什么要分 “有效” 和“无效”
软件不仅要能接收合理的数据，也要能经受意外的考验；因此既要测有效数据，也要测无效输入。

有效等价类验证系统在合法输入下是否能正确实现功能；无效等价类用于验证系统对非法输入能否正确拒绝、报错或保护。

-----
** 常见划分规则
*** 第一类：输入有效值范围
比如年龄 18~60、成绩 0~100。

一般可以划分为：
- 一个有效等价类：范围内
- 两个无效等价类：高于范围、低于范围
*** 第二类：输入必须属于某个集合
比如支付方式只能是“微信/支付包/银行卡”

可以划分为：
- 一个有效等价类：属于该集合
- 一个无效等价类：不属于该集合
*** 第三类：输入必须满足某种规则
比如用户名必须字母开头、后接字母数字、长度不超过6等。

可以划分为：
- 一个有效等价类：符合规则
- 多个无效等价类：从不同角度违反规则
*** 第四类：布尔量
比如“是否同意协议”

- 有效类：True
- 无效类：False


是否把False当作无效，要看业务语义。
** 等价类设计步骤

先数据分类，分有效类与无效类。再对有效类进一步分割，形成等价类表并编号；然后为每个有效等价类设计测试用例，每个无效等价类也设计测试用例。

1. 找输入条件
2. 按规则划分类别
3. 给每类编号
4. 每类挑代表值并写预期结果


-----
* 等价类的典型例题
** 例题1：成绩输入框
输入范围0~100，整数。

应该这样划分：

有效等价类：

- E1：0~100的整数


无效等价类：
- E2：小于0
- E3：大于100
- E4：非整数，如88.5
- E5：非数字，如abc
- E6：空值


对应的测试用例：

- 60→接受，正常保存
- -1→拒绝，提示范围错误
- 101→拒绝，提示范围错误
- 88.5→拒绝，提示必须为整数
- abc→拒绝，提示格式错误
- 空→拒绝，提示不能为空

** 例题2：用户名规则题

#+BEGIN_EXAMPLE
用户名用字母开头，后跟字母或数字任意组合，长度不超过6
#+END_EXAMPLE

可以这样分：

有效等价类：

- E1：字母开头，后续为字母/数字，总长度为1~6


无效等价类：

- E2：不是字母开头
- E3：包含非法字符
- E4：长度为0
- E5：长度超过6


测试用例：
- ~abc12~ → 有效
- ~1abc~ → 无效
- ~a_bc~ → 无效
- ~空字符串~ → 无效
- ~abcdefgh~ → 无效

** 例题3：电话号码

比如：

#+begin_example
区号是已0开头的3为或4位数字，号码是以非0或非1开头的7位或8位数字
#+end_example

这里需要拆成两个字段：

- 字段一：区号
- 字段二：号码


区号有效类：

- A1：0开头3位
- A2：0开头4位


区号无效类：
- A3：不是0开头
- A4：长度不是3位或4位
- A5：含非数字


号码有效类：
- B1：首位2-9，长度7位
- B2：首位2-9，长度8位


号码无效类：
- B3：首位为0或1
- B4：长度不是7位或8位
- B5：含非数字


测试用例就用“区号+号码”的组合来写。

** 例题4：日期校验

#+begin_example

* 边界值分析法
** 边界值是什么
边界值分析法就是对输入或输出的边界进行测试。通常它是等价类划分的补充，测试用例来自等价类的边界。
** 边界值怎么取
如果输入有边界，就取：
- 正好等于边界
- 刚刚小于边界
- 刚刚大于边界


如果输入规定个数，就取：
- 最小个数
- 最小个数-1
- 最大个数
- 最大个数+1


如果输入是有序集合，就取：
- 第一个元素
- 最后一个元素


-----
* 边界值的典型例题
** 年龄18~60

边界值：

- 17
- 18
- 19
- 59
- 60
- 61

* 因果图法
** 为什么需要因果图
等价列和边界值主要着重考虑输入条件本身，但不太会考虑输入条件之间的组合和相互制约；当多个输入条件之间存在依赖关系时，就需要因果图。

当题目出现“如果...并且...否则...”这类组合逻辑时，就需要因果图。
** 因果图在干什么
把输入条件（原因）和输出动作（结果）之间的逻辑关系画出来。
** 因果图的设计步骤
1. 分析规格说明，列出输入和输出，并编号
2. 分析输入之间的关系
3. 分析输入和输出之间的逻辑关系
4. 标注不可能组合和约束，形成因果图
5. 由因果图转化为决策表
6. 决策表每一列对应一个测试用例


因果图本质上是“从文字逻辑到决策表”的中间桥梁。
------
* 决策表法
** 什么是决策表
决策表由“条件”和“动作”两部分组成。

- 条件桩：列出所有条件
- 动作桩：列出所有动作
- 条件项：每个条件在某条规则里的取值
- 动作项：在该条件组合下执行哪些动作
- 每一列是一条规则


决策表是用表格把“什么条件下做什么动作”列清楚。
** 什么时候用决策表
题目满足这三个特征：
- 条件有多个
- 条件之间有组合
- 不同组合对应不同动作
** 决策表怎么做
1. 列条件
2. 列动作
3. 列出可能组合
4. 删除不可能或无意义组合
5. 合并动作相同的规则
6. 每列生成一个测试用例
* 决策表的典型例题
** 例题1：机器维修

#+begin_example
功率大于100马力的机器、维修记录不全或已运行6年以上的机器，应给予优先维修处理。
#+end_example

条件：
- 功率是否大于100马力
- 维修记录是否完整
- 运行是否超过6年


动作：

- 优先维修
- 普通维修


决策表为：

| 序号            | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|-----------------+---+---+---+---+---+---+---+---|
| 条件            |   |   |   |   |   |   |   |   |
| 功率大于100马力 | Y | Y | Y | N | N | Y | N | N |
| 运行记录完整    | Y | Y | N | Y | N | N | Y | N |
| 运行超过六年    | Y | N | N | Y | Y | Y | N | N |
|-----------------+---+---+---+---+---+---+---+---|
| 动作            |   |   |   |   |   |   |   |   |
| 优先维修        | 1 | 1 | 1 | 1 | 1 | 1 | 1 |   |
| 普通维修        |   |   |   |   |   |   |   | 0 |
* 必须记住的

1. 黑盒测试不能穷举，所以要选代表性数据
2. 等价类把输入空间分块，每类选代表
3. 等价类必须同时考虑有效类和无效类
4. 边界值专门测边界及其附近容易出错的位置
5. 输入条件之间有逻辑组合关系时，优先使用决策表法和因果图法
6. 因果图分析逻辑，决策表落地为测试用例
